My Approach to AI

AI reads. Rules verify. Humans decide.

This isn't specific to rebate reconciliation. This is how I think about AI in any finance workflow. Three layers, every time.

AI

Reads and extracts

AI parses unstructured documents — remittances, PDFs, spreadsheets. It follows a defined schema: what fields to find, in what order, with what rules. Every extracted value includes a source citation so the human can verify without re-reading the whole document. This is called schema-guided extraction.

Rules

Checks the math

No AI here — pure deterministic logic. Do the line items sum to the total? Does the manufacturer name match a known entity? Is this a duplicate? Is the amount within sanity bounds? If any check fails, the human sees exactly what failed and why. The system never silently passes through bad data.

Human

Makes every decision

Every AI output has a confirm, edit, or reject control. Nothing executes without approval. The system tracks how often humans correct the AI. Over time, fields that are never corrected earn auto-confirmation. Fields that get corrected stay highlighted for review. Trust is earned through measured accuracy, not promises.

Why this matters: Raw AI extraction is ~95-97% field-level accurate. Not good enough for finance. But you don't need the AI to be perfect — you need the system to be perfect. Layer 2 (deterministic rules) catches what AI misses. Layer 3 (human review) catches what rules miss. The combination is what makes it production-grade.

Week One

What I need to learn from you

These aren't generic questions. Each one changes the design in a specific way.

Process

"Walk me through a single reconciliation cycle — what happens from claims export to client pass-through?"

Determines the full workflow scope and where automation boundaries should sit.

Pain

"What's the most time-consuming part — the matching, the investigation, or the reporting?"

Decides which step gets automated first. If matching is already fast, I focus elsewhere.

Data

"What does a typical remittance look like? Detailed line items, or a lump sum with no breakdown?"

Determines whether AI document parsing is even needed, or if the problem is purely a matching logic issue.

Scope

"What does Plenful handle vs. what's still manual?"

I need to know the boundary so I don't build something that already exists.

Scale

"How many manufacturers per cycle, and how many hours does it take?"

Sizes the ROI. 5 manufacturers and 10 hours is a different project than 30 manufacturers and 80 hours.

Migration

"What's the timeline for moving to homegrown adjudication only?"

Anything I build needs to survive this transition. If it's 3 months away, I design differently than if it's 18 months.
